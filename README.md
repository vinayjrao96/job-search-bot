# job-search-bot

An automated job search pipeline that runs on GitHub Actions every hour. It scrapes job postings across LinkedIn, Indeed, Glassdoor, Naukri, and other boards — filtered for India-based roles, remote roles that accept India timezone, and international roles with visa sponsorship — scores each role against your resume using Gemini AI, generates tailored resume bullets and a cover letter for strong matches, and emails you a brief summary with the full listing attached as a CSV.

It never auto-submits applications. Every email gives you the apply link and tailored materials so you can review and act yourself.

---

## How it works

```
python bootstrap.py        ← run once (or after updating resume.txt)
        │
        ▼ profile.json (search terms, keywords, location — all resume-derived)
        │
GitHub Actions (hourly cron)
        │
        ▼
  fetch_new_postings()
  ├── Apify  → LinkedIn, Indeed, Glassdoor, Naukri    (primary, pay-per-result)
  ├── JSearch → Google for Jobs aggregator            (free tier fallback)
  ├── Greenhouse API → direct-to-company              (free, unlimited)
  └── Lever API      → direct-to-company              (free, unlimited)
        │
        ▼  keyword filter  →  date filter (POSTED_WITHIN_DAYS)
           location filter (India / remote-India / visa sponsorship)
           sort by posted date (most recent first)
        │
        ▼  deduplicate against jobs.db
        │
        ▼  Gemini (score 0–100 vs resume + location/visa context)
        │
        ├── score < 70  → store, skip email
        └── score ≥ 70  → generate tailored bullets + cover letter
                               → email: brief summary note + jobs_digest.csv attached
```

## Setup

### 1. Clone and install

```bash
git clone https://github.com/<your-username>/job-search-bot.git
cd job-search-bot
python -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
```

### 2. Add your resume

Replace `resume.txt` with your resume in plain text. All scoring, material generation, and search term derivation is driven by this file.

### 3. Get your API keys

| Key | Service | Cost | How to get |
|-----|---------|------|------------|
| `GEMINI_API_KEY` | Google AI Studio | Free (~20 req/day) | [aistudio.google.com/apikey](https://aistudio.google.com/apikey) |
| `APIFY_API_KEY` | Apify | $0.003 / job scraped | [console.apify.com/account/integrations](https://console.apify.com/account/integrations) |
| `RAPIDAPI_KEY` | JSearch via RapidAPI | Free (200 req/month) | [rapidapi.com → JSearch → Subscribe (Basic)](https://rapidapi.com/letscrape-6bRBa3QguO5/api/jsearch) |
| `GMAIL_ADDRESS` | Gmail sender address | Free | Your Gmail address |
| `GMAIL_APP_PASSWORD` | Gmail app password | Free | [myaccount.google.com/apppasswords](https://myaccount.google.com/apppasswords) — requires 2FA |

### 4. Configure environment

```bash
cp .env.example .env
# Fill in your five keys
```

### 5. Bootstrap from your resume

```bash
python bootstrap.py
```

This makes a single Gemini call, extracts your target titles, skills, location, search terms, and keywords from `resume.txt`, and writes them to `profile.json`. `config.py` loads this automatically on every run.

Re-run bootstrap whenever you update your resume. `profile.json` is gitignored — it stays local.

> **What profile.json drives:** `APIFY_SEARCH_TERMS`, `JSEARCH_QUERIES`, `KEYWORDS`, `TARGET_COUNTRY`, `CANDIDATE_LOCATION`, `CANDIDATE_SENIORITY`, `CANDIDATE_SUMMARY`, and `EMAIL_TO`. All of these have hardcoded fallbacks in `config.py` if the file is absent.

### 6. Tune infrastructure settings

Open `config.py` to adjust settings that aren't resume-derived:

| Setting | Default | What it does |
|---------|---------|-------------|
| `POSTED_WITHIN_DAYS` | `7` | Only fetch jobs posted in the last N days. Use 1/7/30/90. |
| `SCORE_THRESHOLD` | `70` | Minimum score to generate materials and email. |
| `APIFY_JOB_BOARDS` | `linkedin, indeed, glassdoor, naukri` | Boards to scrape. |
| `APIFY_MAX_RESULTS` | `20` | Results per board per term. Controls Apify cost directly. |
| `APIFY_INTERVAL_HOURS` | `6` | How often Apify runs. Raise to cut spend. |
| `FILTER_INTERNATIONAL` | `True` | Enforce India/remote/visa rules on international roles. |
| `GREENHOUSE_COMPANIES` | `[]` | Specific company ATS slugs to poll directly. |
| `LEVER_COMPANIES` | `[]` | Specific company ATS slugs to poll directly. |

### 7. Run locally

```bash
source .venv/bin/activate
python main.py
```

## Email digest format

Every run sends one email:

**Subject:** `[Job Digest] {N} strong match(es) of {total} new role(s) — crawled YYYY-MM-DD HH:MM UTC`

**Body:** A brief note with the crawl timestamp, match counts, search window, sources used, and filters applied.

**Attachment:** `jobs_digest.csv` — every scored job with these columns:

| Column | Description |
|--------|-------------|
| Score | Gemini fit score 0–100 |
| Title | Job title |
| Company | Company name |
| Location | Job location |
| Remote | Yes / No |
| Visa Sponsorship | Yes / No (detected from description) |
| Posted Date | Date the job was posted (YYYY-MM-DD) |
| Source | linkedin / indeed / glassdoor / naukri / jsearch |
| Status | strong_match / below_threshold / score_error |
| Apply URL | Direct link to apply |
| Reasoning | Gemini's 2–3 sentence fit summary |

Strong matches also have tailored resume bullets and a cover letter stored in `jobs.db` (retrievable but not emailed inline — keeps the digest clean).

## Deploy to GitHub Actions

### 1. Push to GitHub

```bash
git init
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/<your-username>/job-search-bot.git
git push -u origin main
```

### 2. Add secrets

**Settings → Secrets and variables → Actions → New repository secret** — add all five:

- `GEMINI_API_KEY`
- `APIFY_API_KEY`
- `RAPIDAPI_KEY`
- `GMAIL_ADDRESS`
- `GMAIL_APP_PASSWORD`

### 3. Commit your profile

`profile.json` is gitignored by default (it contains your email). To use it in CI you have two options:

**Option A — add it as a secret** (recommended):
```bash
# Add the contents of profile.json as a GitHub secret named PROFILE_JSON
# The workflow writes it to disk before running main.py
```

**Option B — remove it from `.gitignore` and commit it** (simpler, less private):
```bash
git rm --cached profile.json   # stop tracking if already ignored
# remove 'profile.json' from .gitignore
git add profile.json && git commit -m "Add profile"
```

> Without `profile.json` in CI, the bot falls back to the hardcoded defaults in `config.py` — which still works, just not personalised.

### 4. Trigger a test run

**Actions → Hourly Job Search → Run workflow** fires it immediately without waiting for the cron.

The workflow commits `jobs.db` back to the repo after every run so deduplication persists between hourly executions.

## Cost estimate

| Service | Default config | Monthly estimate |
|---------|---------------|-----------------|
| Apify | 5 terms × 4 boards × 20 results × 4 runs/day | ~$48/month |
| Gemini | scoring + materials calls | Free (within daily limit) |
| JSearch | 2 queries × 4 runs/day | ~240 req/month (free tier = 200 — consider trimming to 1 query) |

**To reduce Apify cost:** lower `APIFY_MAX_RESULTS` to 5–10, trim `APIFY_SEARCH_TERMS` to 2–3, reduce `APIFY_JOB_BOARDS`, or raise `APIFY_INTERVAL_HOURS` to 12 or 24.

---

## How deduplication works

`jobs.db` (SQLite) stores every scored job keyed by a stable ID derived from the source and job URL. GitHub Actions commits this file back after each run so it persists across hourly executions. The `apify_last_run` and `jsearch_last_run` meta keys enforce the rate-limit intervals independently.

## Adding a new job source

1. Add a `fetch_<source>(...)` function in `discovery.py` returning a list of dicts:
   `{id, title, company, location, url, text, posted_date, is_remote, visa_sponsorship, source}`
2. Prefix `id` with a unique short code (e.g. `wb_` for Workable)
3. Call it inside `fetch_new_postings()` with a `try/except` that prints on failure
4. Add any required config or secrets to `config.py` / `.env.example`
